# 부산·대구 청년을 위한 AI 기반 정책·장학금 추천 및 신청 준비 자동화 플랫폼
“마지미” 사업계획서

PSST 구조 기반 초안 | 이미지·도표는 삽입 위치만 표시

## 0. 창업아이템 개요 요약

**아이템명**: 마지미

**한 줄 정의**: 부산·대구 19–29세 대학생·휴학생·취업준비생이 받을 수 있는 정책·복지·장학금을 한 번에 찾고, 놓치지 않게 관리하며, 신청 준비까지 도와주는 원스톱 플랫폼

**핵심 타깃**: 부산·대구 거주 또는 재학 중인 19–29세 대학생·휴학생·취업준비생

**핵심 가치**: 개인 맞춤 추천, 자격 사전판별, 마감일 관리, 서류 체크리스트, 공식 신청처 연결, PRO 자동화

**BM**: FREE(추천·탐색·비교·알림) + PRO(서류 입력 자동화·신청 항목 자동 매핑·제휴처 한정 원클릭 자동 신청) + 제휴 전환 수수료

**MVP 목표**: 비회원 설문 → 추천 미리보기 → 카카오/간편 로그인 → 대시보드 진입 흐름 완성, 부산·대구 데이터 100건 이상 확보, 추천 결과 3초 이내 제공


# 1. Problem — 왜 이 아이템이 필요한지


## 1-1. 시장동향 및 문제 인식

- 청년 대상 정책·복지·장학금은 정부, 지자체, 대학, 재단 단위로 계속 생성·변경되지만 사용자가 이를 한 번에 확인할 수 있는 구조는 부족하다.

- 현재 청년 사용자는 정부 포털, 지자체 청년 플랫폼, 대학 공지, 장학재단 공고를 각각 찾아봐야 하며, 정보 탐색 이후에도 '내가 진짜 받을 수 있는지'를 다시 스스로 판단해야 한다.

- 즉, 시장의 핵심 문제는 정보의 부재가 아니라 '개인별 실행 가능 정보'의 부재이며, 혜택은 존재하지만 실제 신청과 수혜로 이어지는 과정에서 이탈이 크게 발생한다.


## 1-2. 문제점(외적 동기)

- ① 정보 파편화: 정책·복지·장학금이 여러 기관에 흩어져 있어 사용자가 직접 검색·비교·정리해야 한다.

- ② 신청 복잡성: 나이, 지역, 학적, 취업상태, 소득구간 등 조건이 공고마다 달라 자격 판별이 어렵고, 필요한 서류도 항목별로 달라 반복 입력과 준비 부담이 크다.


## 1-3. 문제점(내적 동기)

- ③ 행동 포기와 마감 누락: 사용자는 '나랑 맞는지 모르겠다', '나중에 해야지', '서류가 귀찮다'는 이유로 신청을 미루고, 결국 마감을 놓치거나 중도 포기한다.


> 이미지 삽입 위치 A | 문제 인식 인포그래픽
- 정보가 흩어진 기존 구조(정부/지자체/대학/재단)
- 사용자의 탐색 부담과 마감 누락 흐름
- '존재하지만 받지 못하는 돈' 메시지 시각화


# Problem - 2 | 소비자 Needs · 핵심기능 · 정량적 기대효과


## 1-4. 소비자 Needs

- • 내가 받을 수 있는 것만 먼저 보고 싶다.

- • 지금 무엇을 먼저 해야 하는지 알고 싶다.

- • 부족한 조건과 필요한 서류를 한눈에 확인하고 싶다.

- • 매번 반복 입력하지 않고 빠르게 신청 준비를 끝내고 싶다.


## 1-5. 아이템 핵심기능

- ① AI 맞춤 추천: 지역·나이·학적·취업상태·소득구간 등 프로필을 반영하여 신청 가능성이 높은 항목을 우선 노출

- ② 신청 준비 관리: D-Day, 서류 체크리스트, 신청 상태 추적, 공식 신청처 연결을 통해 '추천 이후 행동'까지 지원

- ③ 자동화 레이어: 서류 보관함, 기본 자동 입력, PRO의 서류 입력 자동화·신청 항목 자동 매핑·제휴처 한정 원클릭 자동 신청 제공


## 1-6. 정량적 기대효과(초기 목표)

- • 비회원 설문 완료 후 추천 미리보기까지의 이탈 최소화

- • 추천 결과 조회율, 공식 신청처 클릭률, 신청 상태 변경률, 7일 재방문율을 핵심 KPI로 관리

- • 서비스 내 예시 사용자 기준 연간 수혜 가능 금액을 4,320,000원 수준으로 시각화하여 행동 유도

- • 초기 MVP 런칭 기준: 부산·대구 데이터 100건 이상, 추천 결과 3초 이내, 관리자 수동 수정 가능, 마감 알림 정상 동작


> 이미지 삽입 위치 B | Needs-기능 매핑 표 + 홈 화면 목업
- '내가 받을 수 있는 게 뭐야?' → 맞춤 추천
- '지금 뭘 해야 해?' → D-Day/체크리스트
- '입력 너무 많다' → 자동화 기능


# 2. Solution — 어떻게 개발해서 차별화할지


## 2-1. 선행 준비현황

- • 제품 정의 완료: 부산·대구 19–29세 대학생·휴학생·취업준비생을 위한 정책·복지·장학금 통합 추천 플랫폼으로 포지셔닝

- • MVP 범위 확정: 온보딩 설문, 맞춤 추천 리스트, 상세 페이지·자격 체크, 마감 알림, 서류 체크리스트, 통합 수혜 대시보드

- • 기술 구조 정의: Next.js + Rust(Axum) + PostgreSQL + Redis + Docker 중심의 웹/PWA 구조 설계

- • 데이터 전략 정의: 공식 API·공식 데이터 우선, 공개 페이지 기반 수동 검수, 제휴/운영자 직접 등록 구조 설계

- • 운영 원칙 정의: 완전 자동 신청보다 정확한 추천, 설명 가능한 룰 엔진, 운영자 보정 가능 구조를 우선


## 2-2. 전체 개발 프로세스

- 데이터 수집(공식 API/공식 포털/수동 검수) → 정규화 및 룰 생성 → 사용자 온보딩 → 하드 필터·룰 매칭·스코어링 기반 추천 → 상세페이지에서 자격/서류/공식 링크 제공 → 신청 상태 추적 및 마감 알림 발송

- AI는 공고문 요약, 자격요건 후보 추출, 추천 이유 문장화에 사용하고, 최종 자격 판정은 룰 엔진이 담당하여 신뢰성을 확보한다.


## 2-3. 최종 산출물 및 차별성

- ① 검색 앱이 아니라 실행 앱: 추천에서 끝나지 않고 준비·신청·관리까지 하나의 흐름으로 연결

- ② 설명 가능한 추천: 왜 추천되는지, 어떤 조건이 맞고 어떤 조건이 확인 필요인지 함께 제시

- ③ 운영 가능한 구조: 자동화보다 데이터 정확도와 운영자 검수 체계를 우선하여 초기 신뢰 확보

- ④ 무료+유료+제휴 구조: 무료 핵심 경험으로 유입을 만들고, 반복 입력 제거·원클릭 자동 신청 등 강한 자동화는 PRO로 분리


> 이미지 삽입 위치 C | 서비스 구조도
- 사용자 앱 / 추천 엔진 / 데이터 수집 워커 / 관리자 페이지
- 추천 카드 → 상세 → 준비함 → 신청현황 흐름


# Solution - 2 | 협약기간 내 사업계획 및 사업비 집행 방향


## 2-4. 협약기간 내 개발 계획(12주 MVP 기준)

- 1단계(Week 1–2): 레포 구조 세팅, Docker Compose, Next.js/Axum 초기 연결, DB migration 세팅

- 3단계(Week 5–6): programs/eligibility_rules/user states 구현, 상세 페이지, 대시보드, 북마크·진행 상태 구현

- 4단계(Week 7–8): 워커 구현, 공식 API 연동, 정규화 파이프라인, 관리자 화면 1차 구현

- 5단계(Week 9–10): 알림 시스템, 추천 설명 문구, 마감일 UX, 신청 도우미 정교화

- 6단계(Week 11–12): 부산·대구 실데이터 정리, QA, 성능 점검, 비공개 테스트 배포


## 2-5. 사업비 집행 방향(안)

- • 인건비: 제품 기획·개발·데이터 운영 중심으로 배치

- • 외주용역비: UI 개선, OCR/문서 처리 보조, 브랜딩/디자인 등 제한적으로 활용

- • 클라우드/개발환경비: 서버, DB, 스토리지, 알림, 로그 관리에 투입

- • 데이터 운영비: 공고 검수, 정규화, 운영자 수동 수정 체계 구축

- • 마케팅비: 캠퍼스 초기 모객, 콘텐츠 제작, 베타테스트 유입 확보

- • 보안·법무·기타: 개인정보 보호, 약관, 파트너십 관련 기본 비용 반영


## 2-6. MVP 산출물 목표

- • 부산·대구 청년 대상 정책·복지·장학금 추천 가능

- • 사용자 프로필 저장 및 맞춤 추천 제공

- • 곧 마감되는 항목, 부족한 서류, 예상 수혜 금액 제공

- • 공식 신청처 이동, 신청 상태 추적, 알림 발송 가능

- • 운영자가 프로그램/룰/마감일을 수정할 수 있는 관리자 기능 확보


> 이미지 삽입 위치 D | 12주 로드맵 간트차트 + 예산 배분 도표


# 3. Scale-up — 어떻게 팔아서 성장할지


## 3-1. 경쟁사 분석 및 상대우위

- 기존 포털형 서비스는 정보 제공과 검색에 강점이 있으나, 개인 자격 사전판별·서류 준비·마감 관리·신청 상태 추적은 약하다.

- 대학 및 기관 공지 채널은 정확도가 높지만 탐색성이 낮고 개인화가 어렵다.

- 마지미는 '맞춤 추천 + 설명 가능한 자격 판단 + 준비함 + 신청 도우미 + 상태 추적'을 하나로 묶어, 추천 이후의 실행 단계를 줄인다는 점에서 상대우위를 가진다.

- 또한 PRO에서는 서류 입력 자동화와 제휴처 한정 원클릭 자동 신청을 통해 명확한 유료 전환 포인트를 확보한다.


## 3-2. 목표시장(TAM-SAM-SOM-LAM)

- • TAM: 청년 정책·복지·장학금 정보를 탐색하고 신청해야 하는 전국 청년층

- • SAM: 부산·대구의 19–29세 대학생·휴학생·취업준비생

- • SOM: 앱 기반으로 초기 전환 가능한 부산·대구 핵심 대학 생활권 사용자

- • LAM: 부산대·경북권 대학·대구권 대학 등 캠퍼스 커뮤니티 중심의 초기 런칭 구간


## 3-3. 비즈니스 모델(수익구조)

- FREE: AI 맞춤 추천, 놓친 돈 계산기, 전체 프로그램 탐색, 혜택 비교, 서류 보관함, 신청 상태 추적, 마감 알림

- PRO: 서류 입력 자동화, 서류 기반 프로필 자동 완성, 신청서 항목 자동 매핑, 제휴처 한정 원클릭 자동 신청, 우선 마감 알림

- 제휴 수익: 금융·주거·교육·서류발급 등 제휴처의 상담·가입·신청 완료 시 전환 수수료 발생

- 핵심 원칙: 제휴 항목은 반드시 [광고/제휴]를 명확히 표시하고, 사용자의 적합도 기준을 통과한 경우에만 노출


> 이미지 삽입 위치 E | 경쟁사 비교표 + BM 다이어그램


# Scale-up - 2 | 사업화 전략, KPI, ESG 실현계획


## 3-4. 사업화 전략(초기 모객 Setting 중심)

- • 대학생 유입 채널: 에브리타임, 학과/학교 커뮤니티, 청년 정책 콘텐츠형 SNS

- • 지역 채널 연계: 부산·대구 청년 플랫폼, 청년센터, 대학 학생처/창업지원단 협력 가능성 확보

- • 사용 장면 중심 메시지: '나에게 맞는 혜택만', '놓친 돈 계산', '서류 준비 한 번에', '마감 전 자동 알림'

- • 전환 전략: 무료 기능으로 신뢰를 확보한 뒤, 신청 직전 시점에 PRO 자동화 기능을 제안


## 3-5. 성과지표 달성전략(Big 4 중심)

- ① 유입: 캠퍼스 커뮤니티 기반 설치/방문 확보

- ② 활성화: 설문 완료율, 추천 결과 조회율, 상세페이지 진입률 관리

- ③ 전환: 공식 신청처 클릭률, 신청 상태 변경률, PRO 체험 전환률 관리

- ④ 유지: D-7/D-3/D-1 알림 클릭률, 7일 재방문율, 신청완료 후 재사용률 관리


## 3-6. 전체 사업계획 및 ESG 실현계획

- E(Environment): 반복 출력·출력물 제출을 줄이는 디지털 신청 준비 경험 제공

- S(Social): 청년층의 정보 비대칭과 복지 사각지대를 줄여 실질 수혜 접근성을 높임

- G(Governance): 제휴 표시 투명성, 설명 가능한 추천, 운영자 검수, 개인정보 최소 수집 원칙 적용

## 5-2. 한 페이지 요약 문안

- 마지미는 부산·대구의 19–29세 대학생·휴학생·취업준비생이 이번 달, 이번 학기에 받을 수 있는 정책·복지·장학금을 한 번에 찾고 놓치지 않게 관리해 주는 실행형 플랫폼이다.

- 기존 서비스가 정보를 나열하는 데 그쳤다면, 마지미는 개인 맞춤 추천, 자격 사전판별, 마감일 알림, 서류 체크리스트, 신청 상태 추적, 공식 신청처 연결을 통해 실제 신청 행동까지 이어지게 만든다.

- 초기 MVP는 정확한 추천과 관리 UX에 집중하고, 이후 PRO에서 서류 입력 자동화와 제휴처 한정 원클릭 자동 신청을 통해 수익화를 검증한다.

- 이를 통해 사용자에게는 수혜 누락 감소와 신청 편의성 향상을, 공급자와 제휴처에는 더 높은 도달률과 전환 가능성을 제공하는 구조를 만든다.
