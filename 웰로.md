> **작성일**: 2026-03-13
> **분석 소스**: Claude (Opus 4.6) + Codex (GPT 5.4) + Gemini — CCG 트라이-모델 합성

## 2. 웰로(Wello) 앱 상세 분석

### 2.1 주요 기능
- 개인 정보·관심사 기반 **맞춤 정책 추천** (프로필 기반 매칭, 점수순 정렬)
- 부처/지자체별 흩어진 정책 **통합 탐색**
- 마감 임박·예정 정책 **실시간 알림** (푸시)
- **가족 프로필** 기반 추천 (부모, 자녀 대리 확인)
- 정책을 쉽게 풀어쓴 **설명 콘텐츠**
- 정부 혜택 외 지역·기업 혜택 연결
- 간편 신청 링크 (해당 기관 사이트 연결)

### 2.2 UX/UI 특징
- **카드형 인터페이스**: 정책 정보를 카드 형태로 나열, 스와이프 탐색
- **친근한 톤앤매너**: 공공기관의 딱딱한 용어 대신 이해하기 쉬운 언어
- 슬로건: "세상의 모든 혜택을 담다"

### 2.3 장점
| 항목 | 설명 |
|------|------|
| 추천 UX | 공식 포털보다 "나에게 맞는 것만 먼저 보여주는" 경험 |
| 설명력 | 정책 원문 나열이 아닌 쉬운 풀이 |
| 가족 확장성 | 본인 + 부모/가족 정책까지 관리 |
| 업데이트 속도 | 민간 앱답게 버그 수정·기능 추가 빠름 |
| 파편화 해결 | 중앙정부 + 지자체 정책을 한데 큐레이션 |

### 2.4 단점
| 항목 | 설명 |
|------|------|
| 신청 원스톱 부재 | 정책 안내까지만, 실제 신청은 외부 사이트로 이동 필요 |
| 행정 데이터 미연동 | 사용자가 수동으로 정확히 입력해야 함 (마이데이터 X) |
| 공신력 한계 | 정부 공식 서비스 대비 신뢰도·인지도 싸움 필요 |
| 안정성 이슈 | 가족 추가 시 멈춤 등 오류 리뷰 존재 |
| 리뷰 표본 소규모 | App Store 평점 3.4/5 (45개 평가) |

### 2.5 사용자 평판 요약
- **긍정**: 몰랐던 정책 발견, 육아/가족 정책 편리, 디자인 좋음
- **부정**: 일부 화면 오류/먹통
- **종합**: 방향성은 호평, 제품 안정성과 대중적 신뢰 확장은 숙제

## 7. 핵심 차별화 전략 (3축)

### 전략 1: "챗봇으로 30초 만에 내 정책 찾기" (AI-First UX)
- **기존 앱**: 필터/카테고리 나열 → 사용자가 찾아야 함
- **우리 앱**: 자연어 대화로 상황 파악 → AI가 찾아줌
- **기술**: LLM 기반 대화형 인터페이스 + RAG(정책 DB 검색) + 설명 가능한 AI
- **벤치마크**: Benefits.gov 질문-답변형 + 토스 간편 UX
- **Codex 추가 제안**: 추천 결과에 "왜 추천됐는지" 근거를 붙이는 설명형 추천

### 전략 2: "놓치는 돈 0원" 정책 매니저 (지속적 관계)
- **기존 앱**: 일회성 검색 도구
- **우리 앱**: 개인 정책 매니저 — 캘린더, 알림, 갱신 관리, 생애주기 트래킹
- **핵심**: 앱을 "한 번 쓰고 끝"이 아닌 "계속 켜놓는" 서비스로
- **수혜금액 시뮬레이터**로 "이만큼 받을 수 있다"는 동기 부여
- **Codex 추가 제안**: 예산 소진/조건 변경 감지하는 "정책 체인지 디텍션" 엔진

### 전략 3: "정보→서류→신청 원스톱" (실행까지 책임)
- **기존 앱**: 정보 안내에서 끝, 신청은 외부로 이동
- **우리 앱**: 서류 체크리스트 자동생성 + 양식 자동 채우기 + 딥링크 신청
- **마이데이터/공공API** 연동으로 수동 입력 최소화
- **커뮤니티 후기**로 "이 정책 실제로 어떤지" 사회적 검증
- **Codex 추가 제안**: 홈을 `지금 신청 가능 / 곧 열림 / 조건 부족` 3탭으로 단순화

## 10. MVP 핵심 기능 5가지

| 우선순위 | 기능 | 설명 |

| 1 | **퍼스널 프로필** | 나이, 지역, 학력, 취업상태, 소득, 가구형태 입력 (1분 온보딩) |

## 11. 결론 & Next Steps

### 3모델 합의 사항
| 항목 | 결론 |
|------|------|
| **최대 기회** | 기존 앱들은 "정보 나열"에 머무름 → **"AI 대화 + 실행까지 원스톱"**이 블루오션 |
| **킬러 기능 3개** | (1) AI 챗봇 대화형 추천 (2) 수혜금액 시뮬레이터 (3) 정책 조합 최적화 |
| **UX 방향** | 공공앱이 아닌 **토스/카카오뱅크 수준의 민간 금융앱 UX** 지향 |
| **포지셔닝** | 정부24/복지로와 정면승부 X → **청년 특화 + 설명 가능 추천 + 행동 관리**에 집중 |
| **MVP 철학** | "정책을 많이 모으는 것"보다 **"내게 맞는 정책을 바로 행동하게 만드는 것"** |

### Next Steps
1. **정책 데이터 수집 방법 확정** — 공공 API vs 크롤링 vs 수동 입력
2. **와이어프레임/프로토타입** 제작 — 온보딩 → 피드 → 상세 → 신청 플로우
3. **기술 스택 결정** — 프론트엔드(Flutter/React Native), 백엔드, AI/LLM 선택
4. **정책 DB 구축** — 최소 100개 정책으로 MVP 테스트
5. **사용자 인터뷰** — 청년 10명 pain point 검증
