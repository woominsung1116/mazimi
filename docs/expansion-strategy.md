# 마지미 확장 전략 — "자격 판단 엔진" 회사

## 핵심 포지셔닝

> 마지미는 "정책 추천 앱"이 아니라 **"자격 판단 엔진을 가진 회사"**이다.
>
> 우리의 진짜 자산: **JSON DSL 룰엔진 + 서류 자동화 + 신청 실행 워크플로우**
> 이 기술이 적용되는 곳이면 어디든 확장 가능.

## 기술 자산

| 자산 | 설명 | 범용성 |
|------|------|:---:|
| JSON DSL 룰엔진 | 복잡한 자격 조건을 코드로 판단, 가능성 % 계산 | ★★★★★ |
| 서류 자동 체크리스트 | 필요 서류 + 발급처 + 소요시간 + 준비 순서 | ★★★★★ |
| 신청 실행 워크플로우 | 웹뷰 자동채우기 + 클립보드 + 상태 추적 | ★★★★ |
| 7개 소스 자동 수집 | 공공API + 스크래핑 + 수동 데이터 파이프라인 | ★★★★ |
| 디지털 서류 보관함 | 서류 업로드 + 만료 알림 + 혜택 매칭 | ★★★★ |

## 확장 영역 상세

---

### 1. 중소기업 정부지원사업 매칭

**시장**: 연간 정부 중소기업 지원 예산 20조원+, 실제 수혜율 30% 미만

**현재 경쟁**:
- 보조개: AI 요약 + 매칭률 표시 (실행 없음)
- 비즈넵: 세금/회계 중심 (지원사업 약함)
- 행정사: 건당 50-200만원 (비쌈)

**마지미 차별점**:
- 가능성 % 계산 (룰엔진)
- 서류 자동 체크리스트 (발급처+소요시간)
- 신청 단계별 가이드 + 상태 추적
- 보조개가 "요약"에서 끝나는 곳에서 "실행"까지 감

**기술 변경**: 거의 없음. program_type = 'sme_subsidy' 추가 + 사업자 프로필 필드(업종/매출/직원수)만 추가

**수익 모델**:
- 무료: 매칭 + 가능성 % 확인
- 유료 (월 5-10만원): 서류 체크리스트 + 마감 알림 + 신청 가이드
- 프리미엄 (건당 30-50만원): 행정사 매칭 대행

**시작 시점**: 유저 1만명 후 (청년 앱에서 검증된 기술을 그대로 이식)

---

### 2. HR 복지 자동화

**문제**: 기업 직원들이 받을 수 있는 정부지원이 수십 가지인데 아무도 안 챙김
- 청년내일채움공제, 육아휴직급여, 출산급여
- 고용안정장려금, 직업능력개발
- 근로장려금, 자녀장려금
- 중소기업 취업 소득세 감면

**현재**: HR팀이 수작업 안내하거나 아예 안 함

**마지미가 하면**:
```
기업이 직원 정보(나이/연봉/가족) 일괄 업로드
→ 룰엔진이 직원별 받을 수 있는 혜택 자동 매칭
→ 직원에게 알림 "OOO님, 이번 달 신청 가능한 정부지원 3건"
→ 서류 체크리스트 + 신청 가이드 + 상태 추적
```

**기술 변경**: 기업 관리자 대시보드 + 일괄 프로필 업로드 API

**수익 모델** (직원 수 기반 SaaS):
- 50인 미만: 월 20만원
- 50-300인: 월 50만원
- 300인+: 월 100만원+

**왜 현실적**: 연말정산 솔루션처럼 "있으면 편하고 없으면 귀찮은" 도구. HR SaaS 시장 검증됨 (Flex, 쉬프티).

**시작 시점**: 유저 5만명 후 (B2B 영업 레퍼런스 필요)

---

### 3. 이민/비자 자격 판단

**시장**: 국내 체류 외국인 250만명, 매년 비자 갱신/변경

**현재**: 행정사 상담 건당 30-100만원, 출입국관리사무소 반나절 대기

**룰엔진 그대로 적용 가능**:
```json
{
  "all": [
    {"field": "visa_type", "op": "in", "value": ["E-7", "E-9"], "weight": 30},
    {"field": "stay_duration_months", "op": "gte", "value": 12, "weight": 20},
    {"field": "annual_income", "op": "gte", "value": 30000000, "weight": 25},
    {"field": "criminal_record", "op": "eq", "value": false, "weight": 25}
  ]
}
```

**기술 변경**: 코드 변경 0. 프로필 필드와 룰 데이터만 추가.

**수익 모델**:
- 무료: 자격 확인 + 필요 서류 목록
- 유료 (건당 5-10만원): 서류 체크리스트 + 신청서 자동 채우기
- 프리미엄 (건당 30-50만원): 행정사 매칭 대행

**리스크**: 비자 규정 자주 변경, 틀리면 심각 → 면책 조항 필수 + 행정사 검수 연계

**시작 시점**: 유저 5만명 후

---

### 4. 보험/대출 자격 사전심사

**왜 제일 돈이 되나**: 금융사가 실제로 돈을 냄.

**현재 문제**:
```
고객 → 은행 앱 → 대출 신청 → 심사 → 50%가 거절 (시간 낭비)
```

**마지미가 하면**:
```
고객 → 마지미 사전심사 → "승인 가능성 92%"
     → "A은행 93%, B은행 87%, C은행 72%"
     → "이 서류만 준비하면 됨"
     → 은행 연결
```

**금융사가 돈 내는 이유**: 사전 검증된 고객 = 승인률 높은 리드 = 심사 비용 절감

**CPA 단가**:
| 상품 | 판단 조건 | 건당 |
|------|----------|:---:|
| 전세대출 | 소득/DTI/LTV/무주택 | 10-15만원 |
| 신용대출 | 신용점수/소득/재직기간 | 3-5만원 |
| 청년도약계좌 | 나이/소득/가입이력 | 3만원 |
| 보험 | 나이/직업/건강상태 | 2-5만원 |

**기술 변경**: 없음. 같은 룰엔진으로 금융 상품 자격 판단.

**시작 시점**: 유저 10만명+ (금융사 제휴는 규모 필요)

## 확장 로드맵

```
현재              6개월 후           12개월 후           18개월 후+

청년 정책     →  + 전세대 확장   →  + 소상공인 지원  →  + 보험/대출 사전심사
(부산/대구)      + 대학 SaaS        + HR 복지 자동화     + API/SDK 판매
                 + 세무/환급         + 이민/비자          + "자격 판단" 인프라
```

## Wello vs 마지미 확장 비교

| | Wello | 마지미 |
|---|---|---|
| 확장 방향 | 더 많은 데이터 → 광고/API 판매 | 더 깊은 실행 → 더 많은 산업 |
| 수익 축 | 데이터 유통 (웰로API/웰로애즈) | 프로세스 자동화 (SaaS/CPA) |
| 해자 | 2,700개 기관 데이터 | 룰엔진 + 서류 워크플로우 + 실행 UX |
| 한계 | 정보 플랫폼 → 광고 의존 | 실행 플랫폼 → 산업별 확장 |

## 투자자 스토리

> "마지미는 청년정책 추천 앱이 아닙니다.
> **'누가 무엇을 받을 수 있는지' 판단하고, 실제 신청까지 끝내게 만드는 자격 판단 엔진**입니다.
> 청년은 시작점이고, 같은 엔진이 소상공인 지원금, HR 복지, 비자, 보험/대출까지 확장됩니다.
> 최종적으로는 **대한민국 모든 행정 신청의 자동화 인프라**가 됩니다."
