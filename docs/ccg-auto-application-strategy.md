# 마지미 자동 신청 워크플로우 — 제약 돌파 전략 (CCG 합성)
> 2026-03-21 | Claude + Codex(GPT-5.4) + Gemini

## 핵심 원칙
정부24를 "대체"하지 않고, 위에 "실행 레이어"를 얹는다.
토스/삼쩜삼 모델: 사용자가 인증 → 우리가 나머지 자동화.

## 제약별 돌파

| 제약 | 해결 |
|------|------|
| 정부24 API 없음 | 사용자 직접 발급 → 업로드/전자문서지갑 수신 |
| CODEF 비쌈 | 틸코블렛(스타트업 친화) or 공공 마이데이터 이용기관 신청 |
| 공동인증서 | 간편인증(카카오/PASS) 사용, 공동인증서 저장 안 함 |
| 대리 신청 법적 | "신청 어시스턴트" — 최종 제출은 사용자가 직접 (삼쩜삼 판례) |
| 양식 다름 | 웹뷰 JS 주입 자동채우기 + 클립보드 복사 fallback |

## 3단계 로드맵

### Step 1 (지금, 0원)
- 프로필 → PDF 자동생성 (pdf-lib)
- 개인정보 원터치 클립보드 복사
- 정부24/복지로 웹뷰 열기

### Step 2 (1-3개월, 0원)
- 웹뷰 + evaluateJavascript 자동채우기
- dispatchEvent로 키보드 입력 시뮬레이션
- 최종 제출은 사용자가 직접

### Step 3 (3-6개월, 법인 필요)
- 공공 마이데이터 이용기관 신청 (mydata.go.kr)
- 전자문서지갑 연계 (59개 민간앱 연계 중)
- 사용자 동의 하에 API로 서류 직접 수신

## 법적 안전선 (삼쩜삼 판례)
- ✅ 정보 자동 입력 = 도우미
- ✅ 서류 자동 첨부 = 도우미
- ✅ 신청서 미리 채우기 = 도우미
- ❌ 최종 제출 대신 누르기 = 대리 (위법)

## CODEF 대안
- 틸코블렛 (TilkoBllet) — 공공기관 스크래핑 특화, 스타트업 친화
- 바로써트/링크허브 — 간편인증 중계만 (저렴)
- 공공 마이데이터 이용기관 — 무료, 보안 심사 3개월
