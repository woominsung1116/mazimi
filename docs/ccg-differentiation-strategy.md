# 마지미 차별화 전략 (CCG 합성 결과)
> 2026-03-21 | Claude + Codex(GPT-5.4) + Gemini 3모델 합성

## 핵심 포지셔닝
> 마지미 = "부산/대구 청년의 생애 상태를 이해하는 실시간 혜택 운영체제"
> 정부 서비스가 "정보 저장소"라면, 마지미는 "상태 변화 → 맞춤 추천 → 신청 실행"까지 연결하는 플랫폼

## Top 10 킬러 피처

1. **놓친 돈 계산기** — 온보딩 시 예상 수혜 금액 즉시 표시 (3모델 합의)
2. **조건 변화 자동 알림** — 생일/졸업/이사 시 새 혜택 푸시 (3모델 합의)
3. **추천 이유 + 부족 조건 설명** — 문장으로 "왜 맞는지" 표시 (3모델 합의)
4. **기업 혜택 통합 매칭** — 정부+장학금+민간 한 화면 비교 (3모델 합의)
5. **서류 자동 체크리스트** — 발급처 링크 + 예상 소요시간 (3모델 합의)
6. **수혜 성공도 예측** — 실제 선정 가능성 점수화 (Codex+Claude)
7. **청년 타임라인** — 상태별 혜택 로드맵 (Codex+Claude)
8. **승인 완료 피드 + 수혜 후기** — 익명 실시간 신뢰 구축 (Gemini+Claude)
9. **D-Day 홈화면 위젯** — iOS/Android 위젯 마감 노출 (Gemini+Claude)
10. **신청 난이도 표시** — 상/중/하로 쉬운 것부터 공략 (Gemini+Claude)

## 기업 혜택 B2B 모델

### Codex 제안 (아키텍처)
- 셀프서브 대시보드 → DSL 정규화 → 조건별 타기팅 → 전환 리포트
- B2B 고객: 지역 기업, 대학, 지자체 산하기관, 금융/통신/주거 기업

### Gemini 제안 (UX)
- "총 혜택가 시뮬레이션" — 정부 + 기업 복지 + 정착금 합산
- "역제안 매칭" — 청년 스펙 등록 → 기업이 먼저 제안
- CSR 전용관 — 지역 중견기업 자체 장학금 큐레이션

### 합성 로드맵
1단계: program_type에 corporate_benefit 추가, 같은 룰엔진으로 매칭
2단계: 기업 셀프서브 등록 대시보드
3단계: 총 혜택가 시뮬레이션 + 역제안 매칭

## 수익화 전략 (3모델 합의)

| 단계 | 방식 |
|------|------|
| 초기 | B2C 무료 (청년에게 절대 과금 안 함) |
| 1차 수익 | B2B SaaS 구독 (기업/기관 대시보드) |
| 2차 수익 | 성과형 과금 (지원서 유입당) |
| 3차 수익 | 프리미엄 데이터 리포트 (청년 수요 인사이트) |
| 장기 | 화이트라벨 B2G (대학/청년센터 전용 앱) |

## 부산/대구 지역 특화

- 로컬 핫플 연계 (전포동/동성로 제휴 카페 할인권)
- 지역 산업 맞춤 (부산 해양/물류, 대구 ABB/로봇)
- 대학 거점 QR 마케팅 (부경대/부산대/경북대/영남대)
- 사투리 톤앤매너 옵션
